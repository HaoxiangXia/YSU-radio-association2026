# Repository Guidelines

## Project Overview

`radio-association` is the official information and recruitment management system for the Yanshan University Radio Amateurs Association (燕山大学无线电爱好者协会). It is a small monolithic web app with a vanilla HTML/CSS/JS frontend and a FastAPI backend written in Python, backed by SQLite via the Python standard library `sqlite3`.

Public visitors can browse association info, the about-association page, departments, competitions, trainings, honors, submit an online membership application, and look up admission results. The homepage is a fixed, single-screen landing page with an entrance animation. Recruitment officers can log in via JWT to review, filter, search, and delete membership applications, and export application data as CSV from the dashboard. Admission lookup data is produced separately by a standalone Excel-to-JSON script.

## Domain Language

- `CONTEXT.md` at the repository root is the canonical domain glossary for this project.
- Read `CONTEXT.md` before changing recruitment, application, interview, admission lookup, admission list, or department-assignment behavior.
- Use the terms defined there in product text, code discussions, and new domain-facing names. Prefer `入会申请` over `报名信息`, `招新负责人` over `管理员` when describing recruitment responsibility, and `录取名单`/`录取结果` over implementation names such as `admissions.json`.
- Keep `CONTEXT.md` free of implementation details. It is not a requirements spec, schema design, API contract, or scratch pad; it is only a glossary of domain concepts and avoided terms.
- When a requested change conflicts with `CONTEXT.md`, call out the conflict before editing. If the domain meaning changes, update `CONTEXT.md` at the same time as the code or copy change.

## Architecture & Data Flow

```
Browser (public/html/*.html)
        │
        ▼ static files
   FastAPI static server (public/)
        │
        ▼ REST API
   backend/routes/*.py
        │
        ▼ raw SQL (sqlite3)
   backend/models/*.py
        │
        ▼
   backend/data/database.sqlite
```

- **Entry point:** `backend/app.py` creates the FastAPI app, mounts static files, and includes routers.
- **DB layer:** `backend/config/database.py` opens a shared SQLite database (`backend/data/database.sqlite`, overridable via the `DATABASE_PATH` env var), enables WAL and foreign keys, and creates tables if they do not exist.
- **Routing:** routers are mounted under `/api/*`.
- **Auth flow:** `backend/routes/recruitment_officers.py` validates recruitment-officer credentials from the environment, issues JWTs, and exposes `get_current_recruitment_officer` as a dependency for protected routes.
- **Seeding:** `scripts/init-db.js` is a standalone destructive seed script for the static lookup tables. It does **not** run automatically; you must execute it manually with Bun.

## Development Commands

Backend commands use **uv**:

```bash
# Install Python dependencies and create the virtual environment
cd backend
uv sync

# Start the FastAPI backend (production)
cd backend
uv run uvicorn app:app --host 0.0.0.0 --port 5000

# Dev with hot reload
cd backend
uv run uvicorn app:app --reload --host 0.0.0.0 --port 5000
```

Bun is only required for the standalone JavaScript scripts:

```bash
# Initialize the SQLite database (destructive seed of static data)
bun scripts/init-db.js

# Export admission list from Excel to JSON
bun scripts/export-admissions.js                    # default: 工作簿1.xlsx → public/data/admission-results.json
bun scripts/export-admissions.js input.xlsx out.json
```

Convenience scripts are also defined in `package.json` at the repository root.

## Code Conventions & Common Patterns

### Module System
- Python modules use regular imports; `__init__.py` files are present in `backend/models/` and `backend/routes/`.
- Models are plain Python modules exposing query-helper functions.
- Routes are `fastapi.APIRouter` instances.

### Naming
- Model files: snake_case (`membership_application.py`, `association.py`).
- Route files: snake_case (`recruitment_officers.py`, `membership_applications.py`).
- Internal variables: snake_case for Python (`find_all`, `created_at`, `student_id`).
- Frontend IDs/classes: kebab-case (`membership-application-form`, `recruitment-officer-page`).

### Async & Database
- Route handlers are normal Python functions; database access is synchronous because `sqlite3` is sync.
- Use `get_db()` as a FastAPI dependency to obtain a connection.
- Raw SQL with positional `?` placeholders; prefer prepared statements.

```python
from fastapi import APIRouter, Depends
from config.database import get_db
from models import membership_application as membership_application_model

router = APIRouter()

@router.get("")
def list_membership_applications(db=Depends(get_db)):
    result = membership_application_model.find_all(db)
    return {
        "membership_applications": result["membership_applications"],
        "total": result["total"],
    }
```

### JSON Columns
- JSON arrays are stored as text and parsed manually in models (e.g., `competition.tracks`, `association.awards`).

### Error Handling
- Routes raise `fastapi.HTTPException` with appropriate status codes.
- Unexpected errors are logged and returned as `500` with a clear message.

### Auth Dependency Pattern
- `backend/routes/recruitment_officers.py` exposes `get_current_recruitment_officer` for other routes to reuse:

```python
from routes.recruitment_officers import get_current_recruitment_officer

@router.get("")
def list_membership_applications(
    db=Depends(get_db),
    officer=Depends(get_current_recruitment_officer)
):
    ...
```

### Membership Application Query Pattern
- `membership_application.find_all` accepts query params: `page`, `limit`, `college`, `grade`, `search`, `sort_by`, `sort_order`.
- Sort columns are whitelisted to avoid SQL injection; values are bound with positional params.

## Runtime/Tooling Preferences

- **Runtime:** Python 3.11+ (FastAPI is not compatible with older Python versions).
- **Package manager:** uv (`backend/pyproject.toml` and `backend/uv.lock`).
- **Framework:** FastAPI 0.110+.
- **Database:** SQLite 3 via the Python standard library `sqlite3`.
- **Authentication:** JWT (`PyJWT`) with PBKDF2 password verification; only PBKDF2 hashes are accepted.
- **Environment variables:** `PORT` (default `5000`), `JWT_SECRET` (required; no hard-coded fallback), `RECRUITMENT_OFFICER_ACCOUNTS` (format `username:password:name;...`; required), and `DATABASE_PATH` (optional; relative paths resolve against the repository root, default `backend/data/database.sqlite`). `dotenv` is loaded in `backend/app.py`, but no `.env` file is committed; create one locally.
- **No linting or formatting tooling** is configured (no ESLint, Prettier, Ruff, Black, etc.).
- **No CI/CD** is configured (no `.github/`, `.gitlab-ci.yml`, etc.).
- **CORS** is not enabled because the static files and API are served from the same origin.

## Testing & QA

- **No automated tests exist.** No `tests/`, `__tests__/`, or `*.test.*` / `*.spec.*` files were found.
- **No test framework** is installed.
- **No linting, formatting, or pre-commit hooks** are configured.
- **No CI/CD pipeline** is configured.

Current QA relies on manual verification against the running FastAPI server. If you add tests or linting, update `package.json` and/or `backend/pyproject.toml` scripts accordingly and document the commands in this file.
